## Detail

첫 번째 회사에서는 SI,SM 프로젝트의 경험이 대부분 이지만, 두 번째 회사에서는 디지털사업본부로 들어가 좀 더 기술적으로 트랜디하게 프로젝트를 진행하였습니다. 두 번째 회사에서의 대표적인 프로젝트를 기술해보자면 국민은행의 KB부동산 리뉴얼 프로젝트, 클라우드 환경에서 사용하는 통합 모니터링 시스템 Rhea 개발, KB SaaS Platform 인 Bard 개발이 있습니다.
 
'국민은행의 KB 부동산 리뉴얼 프로젝트'는 AI예측시세, 실시간 시세/매물 등 부동산 정보를 제공해주는 KB의 부동산 웹/앱 서비스 입니다. 이 프로젝트에서 저의 주요 역할은 좌표정보를 기반으로 지도에 나타나는 부동산 정보를 제공해주는 API 개발과, 매물/단지/분양 등의 부동산의 상세 정보를 제공해주는 API 개발이였습니다. 
java & spring 기반으로 API를 개발했으며, DB는 MySQL을 사용하였습니다. 애자일 방식으로 프로젝트를 진행했으며, 협업 툴은 jira, confluence, dooray 등을 사용하였고, 매일 스크럼 회의를 통하여 개선사항과 이슈 등을 매일 피드백하며 진행하였습니다.
각종 필터와 검색 조건에 따른 복잡한 비지니스 로직을 개발하면서 많을 것을 배웠습니다. 
또한 실시간으로 지도에 정보를 뿌려주는 서비스기 때문에 성능이 매우 중요하여, 쿼리 튜닝을 통해 좋은 성능을 나오게 하는 것이 저의 역할에서 주요 이슈였습니다. 이슈 해결을 위해 쿼리 튜닝에 힘썼으며, 만약 제 역량에서 부족하면 DBA분의 도움을 받아 진행하였고, jmeter라는 툴을 이용하여 성능 테스트를 하면서 좋은 성능이 나올 때까지 위의 루틴을 반복 진행하였습니다. 운이 좋게 부동산 서비스의 메인 기술 개발 담당하게 되어 많은 것을 배울 수 있던 프로젝트였습니다.

두 번째로, '클라우드 환경에서 사용하는 통합 모니터링 시스템 Rhea 개발'은 KB에서 클라우드 환경화를 진행하는 MSP 사업을 계속 진행중이며, 그 클라우드 환경에서 자원을 통합 모니터링 하는 시스템인 Rhea 라는 프로그램을 개발하는 것을 지칭합니다.
java & spring 기반이고, MSA에 멀티 모듈 구조로 구성되어 있습니다. JPA, Querydsl 를 사용하였으며, 데이터 수집을 도와주는 오픈 소스인 Prometheus, Grafana, PromQL 등이 사용되었습니다. 애자일 방식으로 프로젝트가 진행되었고 협업 툴로는 메타모스트, 레드마인을 사용하였습니다.
저는 이 프로젝트에서 스크럼 마스터 역할을 하며 매일 데일리 스크럼을 주도하였고, 스프린트가 시작하고 끝날 때마다, 스타트 미팅과 회고 미팅을 준비하고 진행하였습니다. 프로젝트의 초기에는 팀장님의 리드하에 초기 환경구성에도 기여하였습니다. Prometheus와 PromQL을 활용 하여 시계열의 매트릭 데이터를 수집하는 배치를 개발하였으며, 자원이 문제가 생길 때 제공하는 alert data를 제공하는 API를 개발하였습니다.
JPA를 많이 사용하여, ORM 사용에 익숙해지는 경험이 되었습니다. 또한 Prometheus라는 오픈 소스를 사용하며 처음에는 많이 힘들었지만 계속 학습하며 잘 활용하여, 새로운 기술을 사용하는 것에 대한 거부감을 깰 수 있는 좋은 경험이 되었습니다. 그리고 초기 환경 구성에 기여하고, 스크럼 마스터를 진행하면서 프로젝트에 대한 전반적인 상황을 파악하며 어떻게 하면 프로젝트를 잘 마무리 할 수 있을까에 대한 고찰을 해볼 수 있는 기회가 되었습니다.

마지막으로 'KB SaaS Platform 인 Bard 개발'은 KB의 어떤 계열사던 KB에서 사용하는 서비스들을 제공해주는 Gateway 역할을 하는 SaaS Platform 개발입니다. java & spring cloud 기반이며, spring gateway, Eureka, Redis, OAuth 2.0, Kafka, R2DBC, WebFlux, MySQL 등의 gateway 관련 여러 기술들이 사용되었습니다.
현재 제가 진행중인 프로젝트이며, 저의 주요 역할은 Auth Server 개발과 DPM, APM 설치와 Redis 성능 개선 등 입니다. spring oauth와 spring security 라이브리를 써서 인증/인가 관련 로직을 구현 하였고, 추가적으로 OAuth 2.0을 참고하여 OAuth 2.0 기능과 비슷하게 토큰을 제공하는 기능을 개발하였습니다. 또한 Gateway Server에서 인증/인가 쪽 filter 구현을 하며 WebFlux와 R2DBC를 사용해보며 non blocking 방식에 대해 조금 이해 할 수 있는 경험이 되었습니다.
 APM으로는 pinpoint, DPM으로는 scouter를 사용하며 설치와 옵션 설정에 기여하였습니다. APM/DPM 환경 구성에 기여하며, docker와 K8s를 사용하며 인프라 영역의 기술 사용에 조금 익숙해지는 경험을 할 수 있었습니다.
